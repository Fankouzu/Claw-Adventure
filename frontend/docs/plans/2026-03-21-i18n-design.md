# Claw Adventure 多语言国际化设计文档

## 概述

为 Claw Adventure Web 添加国际化（i18n）支持，实现英语、简体中文、繁体中文、日语四种语言版本。

## 技术方案

### 选型：next-intl

选择 `next-intl` 作为国际化解决方案，原因：
- Next.js App Router 原生支持
- 自动语言检测与重定向
- SEO 友好的 URL 结构
- 类型安全的翻译 key
- 服务端和客户端组件都支持

## 语言配置

| 代码 | 语言 | 匹配的系统 locale |
|------|------|------------------|
| `en` | English | en, en-US, en-GB, ... |
| `zh-CN` | 简体中文 | zh-CN, zh-SG |
| `zh-TW` | 繁体中文 | zh-TW, zh-HK |
| `ja` | 日本語 | ja |

**默认语言**: `en`

## URL 结构

```
/                    → 重定向到 /en 或根据浏览器语言
/en                  → English 首页
/zh-CN               → 简体中文首页
/zh-TW               → 繁体中文首页
/ja                  → 日本語首页
/en/dashboard        → English Dashboard
/zh-CN/dashboard     → 简体中文 Dashboard
```

## 语言切换器

在 Navbar 右上角添加下拉菜单：


## 文件结构

```
/messages
  /en.json          # English translations
  /zh-CN.json       # 简体中文
  /zh-TW.json       # 繁体中文
  /ja.json          # 日本語

/i18n
  /routing.ts       # next-intl 路由配置

/app

```

## 实现要点

### 1. 中间件配置
- 自动检测浏览器 `Accept-Language` 头
- Cookie 存储用户语言偏好
- 首次访问时重定向到合适的语言版本

### 2. 服务端组件
- 使用 `getTranslations()` 获取翻译
- 在 `generateMetadata()` 中使用翻译

### 3. 客户端组件
- 使用 `useTranslations()` hook
- 语言切换时使用 `useRouter()` 刷新页面

### 4. SEO 优化
- 每个语言版本有独立的 URL
- 在 `<head>` 中添加 `hreflang` 标签
- 正确设置 `<html lang="...">` 属性

## 用户流程

1. 用户首次访问 `/`
2. 中间件检测浏览器语言
3. 如果是 zh-CN/zh-TW/ja，重定向到对应语言路径
4. 否则重定向到 `/en`
5. 用户可通过右上角切换语言
6. 语言偏好保存到 Cookie，下次访问自动应用

## 兼容性

- API 路由 (`/api/*`) 保持不变，不受语言路由影响
- 静态资源 (`/public/*`) 保持不变
- WebSocket 连接不受影响

## 注意事项

1. **metadata 翻译**: 页面标题、描述需要翻译
2. **错误消息**: API 错误消息保持英文（面向 AI Agent）
3. **日期/数字格式**: 按语言习惯格式化（如需要）
4. **图片**: Logo 等通用图片不需要翻译
